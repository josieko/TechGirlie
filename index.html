API, which stands for "Application Programming Interface", is a term you'll frequently encounter in the tech world. Think of an API as a magic gateway that enables different software applications to communicate and share information with each other. Just like in your everyday life, where you connect with friends through conversations, APIs allow software programs to connect and exchange data seamlessly.

Imagine you're using a weather app on your smartphone. The app fetches up-to-the-minute weather data from a weather service, like magic! Well, that magic is made possible by an API.   • Performing actions on a server like when you send an email or create a new user account
  • Building new applications and services like a weather app that uses an API to get the weather forecast info to display it in the app

Short for â€œRepresentational State Transferâ€, a REST API is an architectural style defining a set of constraints to be used for creating web services. It is a way of accessing web services in a simple and flexible way without having any processing.

If you ever thought of making an app that relies on data from a 3rd party online service, you will definitely come across REST since that is the most common way of implementing APIs on the web.

REST APIs mostly communicate with JSON data even though no specification requires request or response data to be in JSON, it became a convention because it is easy to resolve in almost any language.

It is good to know that REST APIs are typically stateless, meaning that each request contains all the information that the server needs to process it. Thatâ€™s also what makes it easy to scale.

GraphQL

Unlike REST, GraphQL is actually a query language allowing clients to request the exact data they need in the format they need it.

In the case of precision, it has a big advantage over REST APIs which often return more data than you need (we all know that one person that overshares) or require you to make multiple requests to get the data you want.

It is also strongly typed and that means you can validate your queries before executing them, which helps prevent errors, ensuring you get the data you expect.

What can be difficult, because GraphQL is relatively new compared to REST, it has a smaller community (that is growing rapidly), fewer tools and libraries available and it can be more complex to design and implement.
